=== Decorator API (RED stub) ===
  export function BatchScheduled(
    cronExpression: string,
    options: BatchScheduledOptions,
  ): MethodDecorator;

  type BatchOverlapPolicy = 'skip' | 'queue' | 'parallel';

  interface BatchScheduledOptions {
    name: string;                  // required, unique
    timezone: string;              // required, IANA
    overlap?: BatchOverlapPolicy;  // optional; default applied at runtime
    startAt?: Date;                // optional, absolute lower bound (by reference)
    endAt?: Date;                  // optional, absolute upper bound (by reference)
    inert?: boolean;               // optional user-facing knob
  }

  interface BatchScheduledMetadata {
    cron: string;
    options: BatchScheduledOptions;
    inert: boolean;                // captured from process.env.BATCH_SCHEDULED_DISABLE === '1'
  }

  Metadata key: BATCH_SCHEDULED_OPTIONS = 'nest-batch:scheduled'
  Stored shape: { cron: <string>, options: <BatchScheduledOptions>, inert: <boolean> }
  Stored on:    descriptor.value (the method function reference), via @nestjs/common SetMetadata
  Read by tests: Reflect.getMetadata(KEY, Job.prototype.run)  // function reference, not (target, key) pair

=== What this evidence captures ===
The 10 happy-path metadata assertions in
  "@BatchScheduled — happy-path metadata is discoverable"
pass against the RED stub. The contract that the decorator MUST set
BATCH_SCHEDULED_OPTIONS metadata that carries cron / name / timezone /
overlap / startAt / endAt verbatim, and MUST NOT leak across sibling
methods, is locked in and is the part of the contract that the
follow-up GREEN implementation (cron/timezone validation) MUST
preserve.

The 3 public-surface assertions in
  "@BatchScheduled — public surface"
also pass (exported constant, function type, decorator factory shape)
but are not exercised by the `-t 'happy-path metadata'` filter.

The 4 inert-mode assertions pass as a bonus because the stub already
captures `process.env.BATCH_SCHEDULED_DISABLE` at decoration time;
that is a pure metadata capture, not a runtime concern. (The
follow-up GREEN task is free to keep this behavior unchanged.)
