    error TS2322: MockInstance<typeof setTimeout> is not assignable to
    MockInstance<(this: unknown, ...args: unknown[]) => unknown>.

These errors are NOT in scope for the packaging-validation task
(which validates the publishable boundary) and have NOT been touched
in this commit. They are recorded here so the next task can pick them
up cleanly. The swc transpile step in every build script still
succeeds, so the runtime .js artifacts are present in dist/ for
core, mikro-orm, typeorm, and bullmq.

The pre-commit `pnpm build && pnpm test` step was therefore NOT run
clean in this commit because the inherited core tsc step is broken.
The pack-dry-run validation itself, which is the actual deliverable
of this task, exits 0 for all four packages with the peer ranges
and file lists documented above.
